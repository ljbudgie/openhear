# audiogram/ — Sovereign Audiogram Data for OpenHear 🦻

Your audiogram is a measurement of your auditory nerve response.  It was
generated by measuring **your body**.  It belongs to you — not to a clinic
database, not to a proprietary fitting application, not to an institution.

This module reads, stores, visualises, and exports audiogram data in the
open `openhear-audiogram-v1` JSON format.

## What This Module Does

| Component        | Purpose                                                    |
|------------------|------------------------------------------------------------|
| `loader.py`      | Load, validate, and analyse audiogram JSON files.          |
| `visualiser.py`  | Render an audiogram in the terminal using Unicode + ANSI.  |
| `export.py`      | Export to CSV, Markdown, or DSP configuration dicts.       |
| `reader.py`      | Read threshold data from a Noahlink Wireless 2 device.     |
| `data/`          | Audiogram data files in openhear-audiogram-v1 format.      |

## The JSON Format

The `openhear-audiogram-v1` format is an open standard — your audiogram
belongs to you, not a clinic database.  Any user can create their own
JSON file from their clinical audiogram.  See
[`data/FORMAT.md`](data/FORMAT.md) for the full specification and a blank
template.

## The Loader → DSP Bridge

The loader feeds directly into the DSP module for personalised audio
processing.  The key function is `get_gain_profile()`, which computes
the amplification needed at each frequency to bring your hearing to the
normal threshold (20 dB HL).  This is the bridge between clinical
measurement and real-time processing.

```
Clinical Audiogram → loader.py → gain profile → dsp/pipeline.py → your ears
```

## Usage

### Loading an audiogram

```python
from audiogram.loader import load_audiogram, get_pta, get_severity

ag = load_audiogram("audiogram/data/burgess_2021.json")

# Pure Tone Average — the single-number summary of hearing loss
pta = get_pta(ag, "right")      # → 72.5
severity = get_severity(int(pta))  # → "severe"
```

### Getting the gain profile for DSP

```python
from audiogram.loader import load_audiogram, get_gain_profile

ag = load_audiogram("audiogram/data/burgess_2021.json")
gains = get_gain_profile(ag, "right")
# [(125, 10), (250, 15), (500, 30), (1000, 55), ...]
# Each tuple: (frequency_hz, gain_db_needed)
```

### Visualising in the terminal

```python
from audiogram.visualiser import print_audiogram

print_audiogram("audiogram/data/burgess_2021.json")
```

Or from the command line:

```bash
python -m audiogram.visualiser audiogram/data/burgess_2021.json
```

### Generating a DSP configuration

```python
from audiogram.export import to_dsp_config

config = to_dsp_config("audiogram/data/burgess_2021.json", "right")
# Returns a dict that maps to dsp/config.py parameters:
# {
#     "gain_profile": [...],
#     "compression_bands": [...],
#     "compression_knee_dbfs": -50.0,
#     "noise_floor_multiplier": 1.4,
#     "voice_clarity_gain": 2.0,
#     ...
# }
```

### Exporting to other formats

```python
from audiogram.export import to_csv, to_markdown

# CSV for spreadsheets
to_csv("audiogram/data/burgess_2021.json", "audiogram.csv")

# Markdown for documentation
md = to_markdown("audiogram/data/burgess_2021.json")
print(md)
```

### Comparing audiograms over time

```python
from audiogram.loader import compare_audiograms

diff = compare_audiograms("audiogram/data/test_2019.json",
                          "audiogram/data/test_2021.json")
# Shows threshold changes per frequency per ear
# Positive = worse, negative = better
```

## The First Dataset

The file `data/burgess_2021.json` is verified clinical data from Dorothy
Thomson Hearing Services (most recent test 20/10/2021).  It serves as the
format reference and the first real dataset in the project.  The
classification confirms a bilateral sensorineural hearing loss with a
high-frequency sloping pattern — mild-to-profound range in both ears.

## Export Functions

The export functions let you take your data anywhere in any format:

- **CSV** — flat table for spreadsheets and data analysis tools.
- **Markdown** — formatted table for documentation and reports.
- **DSP config** — parameters that map directly to `dsp/config.py` for
  personalised real-time audio processing.
