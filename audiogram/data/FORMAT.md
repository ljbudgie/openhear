# openhear-audiogram-v1 — Sovereign Audiogram Data Format

## What This Is

An open, human-readable JSON format for storing audiometric threshold data.
Your audiogram is a measurement of your auditory nerve response — it was
generated by measuring **your body**.  It belongs to you, not to a clinic
database or a proprietary software system.

This format is intentionally simple so it can be read by any tool, any
language, any platform.  No special libraries required.  No vendor lock-in.

## Structure

Every audiogram file is a single JSON object with the following fields:

| Field             | Type   | Description                                              |
|-------------------|--------|----------------------------------------------------------|
| `subject`         | string | Name of the person whose hearing was measured.           |
| `source`          | string | Clinic or facility that performed the measurement.       |
| `date`            | string | Date of the test in ISO 8601 format (`YYYY-MM-DD`).     |
| `notes`           | string | Free-text notes about the test or context.               |
| `format_version`  | string | Always `"openhear-audiogram-v1"` for this version.       |
| `right_ear`       | object | Threshold data for the right ear.                        |
| `left_ear`        | object | Threshold data for the left ear.                         |
| `classification`  | object | Summary classification of the hearing loss (optional).   |

### Ear Objects

Each ear object contains:

| Field        | Type   | Description                                                  |
|--------------|--------|--------------------------------------------------------------|
| `symbol`     | string | Audiometric symbol: `"O"` for right ear, `"X"` for left ear. |
| `thresholds` | array  | Array of threshold measurements (see below).                 |

### Threshold Entries

Each entry in the `thresholds` array:

| Field     | Type | Description                                                   |
|-----------|------|---------------------------------------------------------------|
| `freq_hz` | int  | Test frequency in Hertz (e.g., 250, 500, 1000, 2000, 4000).  |
| `db_hl`   | int  | Hearing threshold level in decibels Hearing Level (dB HL).    |

**dB HL** is the standard clinical unit.  0 dB HL is the threshold of
normal hearing at that frequency.  Higher numbers mean more hearing loss.

### Standard Test Frequencies

Clinical audiograms typically test at:
125, 250, 500, 1000, 1500, 2000, 3000, 4000, 6000, 8000 Hz.

Not all tests include every frequency.  Include only the frequencies that
were actually measured.

### Classification Object (Optional)

| Field         | Type   | Description                                          |
|---------------|--------|------------------------------------------------------|
| `type`        | string | Type of hearing loss (e.g., `"sensorineural"`).      |
| `pattern`     | string | Pattern description (e.g., `"high-frequency sloping"`). |
| `right_pta`   | float  | Pure Tone Average for the right ear (dB HL).         |
| `left_pta`    | float  | Pure Tone Average for the left ear (dB HL).          |
| `right_range` | string | Severity range for the right ear.                    |
| `left_range`  | string | Severity range for the left ear.                     |

## Pure Tone Average (PTA)

The PTA is the mean of thresholds at **500, 1000, 2000, and 4000 Hz**.
These four frequencies cover the speech-critical range and are the standard
basis for classifying hearing loss severity:

| PTA (dB HL) | Severity           |
|-------------|--------------------|
| 0–25        | Normal             |
| 26–40       | Mild               |
| 41–55       | Moderate           |
| 56–70       | Moderately-Severe  |
| 71–90       | Severe             |
| 91+         | Profound           |

## Creating Your Own Audiogram File

1. Get a copy of your audiogram from your audiologist.  You have the right
   to this data — it is a measurement of your body.
2. Copy the template below into a new `.json` file.
3. Fill in your threshold values at each tested frequency.
4. Save it wherever you keep your OpenHear data.

The loader (`audiogram/loader.py`) will validate the file and compute the
PTA and gain profile automatically.

## Blank Template

Copy this, fill in your values, and save as `your_name_year.json`:

```json
{
  "subject": "Your Name",
  "source": "Clinic Name, Address",
  "date": "YYYY-MM-DD",
  "notes": "",
  "format_version": "openhear-audiogram-v1",
  "right_ear": {
    "symbol": "O",
    "thresholds": [
      {"freq_hz": 250, "db_hl": 0},
      {"freq_hz": 500, "db_hl": 0},
      {"freq_hz": 1000, "db_hl": 0},
      {"freq_hz": 2000, "db_hl": 0},
      {"freq_hz": 4000, "db_hl": 0},
      {"freq_hz": 8000, "db_hl": 0}
    ]
  },
  "left_ear": {
    "symbol": "X",
    "thresholds": [
      {"freq_hz": 250, "db_hl": 0},
      {"freq_hz": 500, "db_hl": 0},
      {"freq_hz": 1000, "db_hl": 0},
      {"freq_hz": 2000, "db_hl": 0},
      {"freq_hz": 4000, "db_hl": 0},
      {"freq_hz": 8000, "db_hl": 0}
    ]
  }
}
```

Replace the `0` values with your actual dB HL thresholds.  Add or remove
frequency entries as needed to match what was tested.
